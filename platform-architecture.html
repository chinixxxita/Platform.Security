Platform Architecture

Exploring the structure, security, and risks of modern computing platforms

Layers of a Computing Platform

  • Hardware: Physical components like CPU, memory, and storage.
  • Firmware: BIOS/UEFI, device firmware that initializes hardware.
  • Operating System (OS): Manages resources and enforces security.
  • Applications: Software that provides user services.

Mobile vs. Desktop Platforms

Mobile: Portability, app sandboxing, stricter permissions (Android/iOS).

Desktop: Higher performance, flexible, broader software ecosystem.

Cloud & Virtualization Platforms

Cloud: On-demand services (AWS, Azure), shared responsibility security.

Virtualization: Hypervisors run multiple VMs, boosting efficiency but adding risks like VM escape.

Platform Attack Surface

  • Hardware: Side-channel attacks, tampering
  • Firmware: Rootkits, backdoors
  • OS: Privilege escalation, kernel exploits
  • Applications: Malware, unpatched vulnerabilities

Trusted Platform Module (TPM)

TPM is a dedicated chip that ensures secure boot, stores cryptographic keys, and blocks hidden startup attacks.

UEFI & Chain of Trust

UEFI: Modern BIOS replacement, initializes hardware, supports Secure Boot.

Chain of Trust: Each stage of boot validates the next, preventing tampering.

TPM Versions & Benefits

TPM 1.2: Basic cryptography (SHA-1, RSA)

TPM 2.0: Stronger algorithms, required for Windows 11.

Benefits: Prevents rootkits, bootkits, unauthorized changes.

Limitations

  • Needs hardware support
  • Failure may lock encrypted data
  • Does not block OS-level malware
  • Still vulnerable to physical tampering
